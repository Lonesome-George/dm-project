现在最大的问题是数据读取和计算，相似度计算太耗时间，就算离线计算也很耗时间。
异常情况：
1.比如有的用户会对17773个item进行评分，这种情况我认为不属于正常的用户行为。现在我们决定只取评分高的那一小部分数据进行相似度的计算，我们认为评分低的数据它的相似度只会导致更低的预测评分，这对正确率不会有太大的影响。

结果：
6条记录
20min->7min->3min->2min->1~2sec

使用topK(K=50)的方法计算相似度，会出现很多空缺值，但这似乎并不影响正确率。
使用全计算方式：75%；使用TopK方法：86%。


可以调的参数：
topK(预测的时候用到)
gamma(计算相似度用到)
